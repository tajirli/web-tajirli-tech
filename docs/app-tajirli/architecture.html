Layer Responsibilities

  • Entry: startup, initialization, app lifecycle handling.
  • Navigation: auth gating and role-sensitive routes.
  • Screens: task-specific user interfaces and interactions.
  • State: centralized async actions and domain state.
  • Infrastructure: adapters to API, storage, and realtime.

Architecture Risks

  • Sync polling every 10s can increase backend load.
  • Store growth may create coupling between domains.
  • Auth gate logic must stay aligned with backend policy.
  • Legacy export paths can hide migration debt.
