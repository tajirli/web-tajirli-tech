Overview Architecture Modules + API Progress Tracker Runbook

Local Development

  1. npm install
  2. set VITE_API_URL in .env if backend URL differs
  3. npm run dev
  4. open http://localhost:5173
  5. validate product listing and admin login flows

Release Checklist

  • Run npm run build and ensure dist output is healthy.
  • Verify /products/:slug and /p/:shortId behavior.
  • Confirm admin profile save and password change actions.
  • Confirm pixel IDs are optional and do not break UI.
  • Validate Arabic and French language toggles.

Architecture Update Workflow

Step Action Output
1 Map impacted files and domain boundaries Impact list in progress tracker notes
2 Apply change in feature/core/shared module first Typed or modular implementation
3 Keep backward compatibility in src/api.js if needed No regression for existing JS consumers
4 Test storefront, admin, and analytics paths Confidence before merge/deploy
5 Update .internal-docs pages if architecture shifts Internal docs remain current
