Local Setup

  1. npm install
  2. Copy .env.example to .env
  3. Set JWT_SECRET and baseline CORS_ORIGINS
  4. npm run dev
  5. Validate GET /health and auth login flow

Daily Quality Checks

  • Review startup logs for DB and socket initialization.
  • Smoke test public checkout and protected dashboard routes.
  • Validate webhook route responses in non-prod mode.
  • Verify tracking endpoint write path and retrieval endpoints.
  • Check CORS behavior for mobile and website origins.

Release Checklist

Step Action Expected result
1 Verify env values in target host JWT secret, CORS list, and integration keys are present
2 Start server in clean environment Server starts with healthy database response
3 Run endpoint smoke tests Public and protected route families respond correctly
4 Validate realtime and webhook paths Socket room events and webhook signature flow remain stable
5 Push only to intended remote branch No accidental push to unrelated repositories

Fast Incident Triage

  • Auth failures: inspect token signing secret and token expiry handling.
  • CORS rejections: compare incoming Origin against CORS_ORIGINS.
  • Webhook issues: confirm raw body middleware executes before JSON parse.
  • Realtime drift: verify clients join expected socket rooms after login.
  • Data path errors: run health endpoint and check DB connectivity logs.
