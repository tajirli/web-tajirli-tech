El Basma Docs

El Basma Frontend Internal Documentation Hub

This folder is intentionally isolated from product UIs. Use these pages as an internal dashboard for architecture, module mapping, and delivery progress across web and mobile projects.

Overview Architecture Modules + API Progress Tracker Runbook

Current Stack

  • React 18 + Vite 5
  • React Router 6
  • Axios + modular HTTP client
  • i18next (ar, dz, fr, en)
  • Tailwind CSS + custom utility classes

Domain Areas

  • Public storefront + product detail funnel
  • Admin login/profile/visitor dashboard
  • Order creation + delivery pricing
  • Tracking pixels + visitor analytics
  • OG prerender for social bots

Technical Direction

  • Hybrid mode: legacy JS + new typed modules
  • Feature-first organization in src/features
  • Shared/core abstraction in src/shared and src/core
  • Backward compatibility maintained in src/api.js

Quick Architecture Snapshot

UI Layer: pages + components + contexts
|
Domain Layer: features/products + features/orders
|
Core Layer: core/api/httpClient + core/config
|
Backend APIs: /api/public/*, /api/auth/*, /api/website-settings

How To Use This Folder

  • Open this page directly in browser from local file system.
  • Use Progress Tracker to update sprint status and blockers.
  • Use Architecture and Modules pages when planning refactors.
  • Do not import these files into src/ or public/.
